# AI-Yunxun 项目架构与目标

本项目面向 MCP 工具管理与客户端使用研究，目标是构建一个可扩展的 AI 智能助手。
关注点在“工具管理、调用与可观测性”，保持系统简单可控，避免过度工程化。

## 目标

- 支持添加与管理 MCP 工具（以 Claude Desktop 的 `mcpServers` 标准为主）。
- 以对话为入口，驱动模型调用 MCP 工具完成任务。
- 具备可视化的工具调用状态与日志，便于研究分析。
- 提供“简单模型记忆”：前端生成 `conversationId`，具备持久化聊天记录。

## 边界与原则

- 不建设用户系统与权限体系。
- 不引入复杂的 RAG/向量检索，除非论文需要。
- 以“最少功能满足研究”为原则，优先可用与可解释。

## 系统构成（高层）

1) **Client 应用（核心）**  
负责对话入口、工具管理、状态展示与记忆管理。

2) **内置 MCP Server（当前）**  
ECharts 与 Neo4j 工具服务，供研究与演示使用。

3) **外部 MCP Server（可导入）**  
通过 `mcpServers` JSON 导入 STDIO MCP 服务，作为可扩展能力。

4) **持久化存储**  
MongoDB 作为聊天记忆的持久化存储。

## 核心能力

- **MCP 工具管理**：导入/列出/启停/移除外部 MCP 服务。
- **对话与工具调用**：对话触发工具调用，强调可解释性。
- **调用可观测性**：显示“调用中/成功/失败”与工具来源。
- **简单模型记忆**：保留对话上下文并持久化。

## 关键流程（抽象）

1) 导入 MCP 工具 → 校验配置 → 建立连接  
2) 发起对话 → 模型推理 → 工具调用 → 结果回传  
3) 日志收集 → 解析状态 → 前端展示  
4) 对话记录写入 Mongo → 便于回放与研究复现

## 可选扩展（非当前重点）

- 统一工具规范与结构化日志格式
- 更细粒度的工具分析与统计
- RAG 或向量检索能力（视论文需求）
